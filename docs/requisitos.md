## Requisitos Funcionais

| ID   | Descrição                                                                                                                                                                                                                                                                                                                                                                                          |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF01 | O sistema deve exibir os 23 problemas públicos cadastrados, cada um vinculado ao respectivo Compromisso Decenal da Criança e do Adolescente, permitindo ao usuário navegar entre eles de forma organizada.                                                                                                                                                                                         |
| RF02 | O sistema deve apresentar a estrutura hierárquica do Plano Decenal na seguinte ordem: Problema público → Compromisso → Objetivo → Linhas de Ação → Tarefas, permitindo ao usuário navegar por cada nível da hierarquia.                                                                                                                                                                            |
| RF03 | O sistema deve permitir que todos os usuários visualizem o status de conclusão de cada tarefa (não iniciada, em andamento, concluída) e também o progresso de cada indicador relacionados a ela para acompanhar o progresso da conclusão da tarefa                                                                                                                                                 |
| RF04 | O sistema deve permitir que qualquer usuário realize uma pesquisa textual por palavras-chave, responsável, compromisso ou linha de ação. Os resultados devem ser exibidos na mesma estrutura hierárquica da navegação do sistema exibindo somente os problemas públicos que possuem os termos correspondentes a pesquisa.                                                                          |
| RF05 | O sistema deve permitir que todos os usuários apliquem filtros de compromisso, indicador, período de tempo, responsável, município e região de maneira cumulativa. Os resultados devem ser exibidos na mesma estrutura hierárquica da navegação do sistema exibindo somente todos os problemas públicos que possuam todos os os critérios selecionados pelo filtro.                                |
| RF06 | O sistema deve permitir que qualquer usuário exporte o conjunto de dados de problemas públicos visualizado pelo usuário, filtrado ou não, nos formatos Excel, CSV e PDF, mantendo a fidelidade das informações exibidas na tela.                                                                                                                                                                   |
| RF07 | O sistema deve registrar e exibir, em cada registro de Tarefas, a data, o horário e o nome do usuário responsável pela última atualização realizada.                                                                                                                                                                                                                                               |
| RF08 | O sistema deve manter um histórico completo de todas as atualizações realizadas em uma Tarefa, ordenando cronologicamente da mais antiga para a mais recente, incluindo data, hora e usuário responsável pela alteração. Também deve permitir que todos os usuários autorizados a visualizar o histórico completo das tarefas visualizem o histórico.                                              |
| RF09 | O sistema deve disponibilizar um painel de administrativo que permita ao usuário administrador selecionar perfil, usuário ou secretaria para ativar ou desativar uma permissão.                                                                                                                                                                                                                    |
| RF10 | O sistema deve permitir que qualquer usuário de qualquer parte do sistema selecione uma opção de alto-contraste que altere a exibição do sistema para alto-contraste, garantindo melhor legibilidade para pessoas com deficiência visual ou sensibilidade a contrastes.                                                                                                                            |
| RF11 | O sistema deve exigir autenticação de usuários com permissões elevadas por meio de e-mail e senha, permitindo acesso apenas a usuários devidamente cadastrados e autorizados.                                                                                                                                                                                                                      |
| RF12 | O sistema deve permitir que somente usuários com perfil Conselho Estadual / Secretarias cadastrem novos indicadores por meio de interface gráfica.                                                                                                                                                                                                                                                 |
| RF13 | O sistema deve manter um registro persistente de qualquer alteração referente aos dados dos indicadores, armazenando o identificador do usuário, data e hora da operação e o tipo da alteração (adição, remoção ou alteração), permitindo que somente os perfis Gestor SEDEF e Administrador visualizem o registro.                                                                                |
| RF14 | O sistema deve validar todos os dados inseridos pelos usuários autorizados impedindo a inserção de dados inválidos ou incompletos ao contexto da inserção.                                                                                                                                                                                                                                         |
| RF15 | O sistema deve permitir que todos os usuários visualizem gráficos referentes aos indicadores por meio de um dashboard interativo, selecionando um problema público por vez e visualizando todos os gráficos referentes aos indicadores do problema selecionado.                                                                                                                                    |
| RF16 | O sistema deve disponibilizar um filtro temporal para os gráficos dos indicadores, permitindo que o usuário selecione um ano específico para visualização.                                                                                                                                                                                                                                         |
| RF17 | O sistema deve aplicar diferentes identidades visuais para cada tipo de indicador, mostrando distinção explícita entre cada um.                                                                                                                                                                                                                                                                    |
| RF18 | O sistema deve exibir, no detalhamento de cada Linha de Ação, a identidade do Responsável Principal e entidades Colaboradoras para o avanço de cada tarefa.                                                                                                                                                                                                                                        |
| RF19 | O sistema deve manter o controle de acesso baseado em função, registrando os perfis distintos de: Administrador, SEDEF, Conselho Estadual/Secretarias e público geral. Administradores e SEDEF possuem permissões máximas de administração do sistema, Conselho Estadual/Secretarias podem inserir e editar dados de suas jurisdições e o público geral pode visualizar e exportar dados públicos. |
| RF20 | O sistema deve permitir que somente usuários com perfil Conselho Estadual / Secretarias realizem importação de dados referentes aos indicadores por meio de arquivos CSV, PDF ou XSLX.                                                                                                                                                                                                             |
| RF21 | Os dados, importados ou inseridos, referentes aos indicadores devem ser validados impedindo a inserção de dados inválidos, incompletos ou fora do padrão interno do sistema, com objetivo de evitar falhas ou problemas de compatibilidade.                                                                                                                                                        |
| RF22 | O sistema deve permitir que o perfil Administrador cadastre novos usuários no sistema definindo Email, Senha, Perfil (Gestor SEDEF, Conselho Estadual / Secretaria, Administrador) e Identificador (Nome).                                                                                                                                                                                         |
| RF23 | O sistema deve, após um cadastro de usuário por parte do administrador, enviar para o email recém cadastrado suas credenciais de login (email e senha)                                                                                                                                                                                                                                             |
| RF24 | O sistema deve obrigar o usuário, ao realizar o primeiro login com as credenciais definidas pelo administrador, a alterar a senha para uma senha particular.                                                                                                                                                                                                                                       |

## Requisitos Não Funcionais

| ID | Descrição |
|----|-----------|
| RNF01 | A interface deve ser acessível conforme os princípios do Design Universal, incluindo contraste adequado, configuração de tamanho da fonte e suporte a leitores de tela. |
| RNF02 | O sistema deve ser responsivo (adequado para desktop, tablet e dispositivos móveis). |
| RNF03 | O sistema deve ter capacidade de armazenamento para guardar dados de pelo menos 10 anos. |
| RNF04 | O sistema deve garantir a integridade dos dados exportados (Excel/PDF/CSV), sem perda de informações ou formatações incorretas. |
| RNF05 | O sistema deve ser bem documentado. |
| RNF06 | O design e a experiência devem ser minimamente semelhantes à interface do Conselho Estadual da Criança e do Adolescente. |
| RNF07 | O sistema deve suportar a inserção de um número arbitrário (grande) de dados. |
| RNF08 | O sistema deve encerrar a sessão do usuário automaticamente após 30 minutos de inatividade. |
| RNF09 | O sistema deve utilizar Tokens JWT para aumentar a segurança. |
