| ID   | Prioridade  | Critérios de Aceitação |
| ---- | ----------- | ----------------------- |
| HU01 | Obrigatória | O sistema deverá exibir os 23 problemas públicos cadastrados.<br>Cada problema público deverá estar vinculado ao respectivo Compromisso Decenal da Criança e do Adolescente.<br>O usuário deverá conseguir acessar essas informações sem realizar login.<br>As informações deverão ser apresentadas na estrutura hierárquica: Problema público → Compromisso → Objetivo → Linhas de Ação → Tarefas. |
| HU02 | Obrigatória | O sistema deverá solicitar e-mail e senha para autenticação.<br>Usuários sem autenticação não deverão acessar áreas restritas.<br>Caso as credenciais estejam incorretas, o sistema deverá exibir uma mensagem de erro.<br>No primeiro login com credenciais definidas pelo administrador, o sistema deve obrigar o usuário a alterar a senha. |
| HU03 | Obrigatória | O sistema deve exibir o status de conclusão de cada tarefa (não iniciada, em andamento, concluída).<br>O sistema deve exibir o progresso dos indicadores relacionados a cada tarefa.<br>Todos os usuários devem conseguir visualizar essas informações. |
| HU04 | Obrigatória | O sistema deve apresentar um campo de busca textual para "Palavra-chave", "Responsável", "Compromisso" e "Linha de ação".<br>O usuário deve conseguir combinar dois ou mais campos simultaneamente para refinar a busca.<br>Nenhum dos campos de busca deve ser obrigatório, se o usuário clicar em "Buscar" com tudo em branco, o sistema deve listar todos os registros.<br>Os resultados devem ser exibidos na mesma estrutura hierárquica de navegação do sistema, exibindo somente os problemas públicos que possuam correspondência com o termo pesquisado.<br>Caso nenhum registro atenda aos critérios, o sistema deve exibir: "Nenhum resultado encontrado para os filtros aplicados." |
| HU05 | Obrigatória | O sistema deverá permitir filtro por compromisso, indicador, período de tempo, responsável, município e região.<br>Os filtros deverão poder ser aplicados de forma cumulativa.<br>Os resultados exibidos deverão corresponder à estrutura hierárquica do sistema, exibindo somente os problemas públicos que atendam a todos os critérios selecionados. |
| HU06 | Importante  | O sistema deverá verificar se o usuário está autenticado e possui perfil Gestor SEDEF ou Administrador antes de exibir o histórico completo.<br>O histórico deverá ser exibido em lista ordenada da atualização mais recente para a mais antiga, com informações de data, hora, tipo de ação (adição, remoção, alteração) e nome do responsável pela alteração.<br>Usuários sem permissão não devem ter acesso ao histórico completo. |
| HU07 | Obrigatória | O sistema deverá exibir, em cada tarefa, a data e a hora da última atualização no formato DD/MM/AAAA - HH:MM.<br>Deverá ser exibido também o tipo de ação (adição, remoção ou alteração) e o nome do usuário responsável pela última atualização.<br>Qualquer usuário deve conseguir visualizar essas informações. |
| HU08 | Obrigatória | O usuário deverá ter aplicado filtros ou realizado uma busca antes de exportar.<br>O sistema deverá permitir que o usuário selecione o formato da exportação (Excel, CSV ou PDF).<br>O sistema deverá inicializar o download automático no dispositivo do usuário com os dados correspondentes ao que está sendo visualizado (filtrado ou não), no formato selecionado.<br>A integridade dos dados exportados deve ser garantida, sem perda de informações ou formatações incorretas. |
| HU09 | Desejável   | Deve existir uma opção visível em qualquer parte do sistema para ativar ou desativar o alto contraste.<br>Ao ativar, o fundo do sistema deve mudar para uma cor escura e os textos para cores de alta visibilidade (branco ou amarelo).<br>Se o usuário navegar entre as páginas com o contraste ativo, o modo deve se manter ativado. |
| HU10 | Obrigatória | O sistema deve disponibilizar um painel administrativo acessível somente a usuários com perfil Administrador ou Gestor SEDEF.<br>O administrador ou gestor SEDEF deve conseguir visualizar os usuários cadastrados e alterar seus níveis de permissão: Alterar tarefas, Gerenciar usuários, Carregar dados, Visualizar dados.<br>Usuários sem permissão não deverão conseguir acessar o painel de gerenciamento. |
| HU11 | Obrigatória | Apenas usuários com perfil Conselho Estadual / Secretarias podem visualizar o formulário de cadastro de indicadores e salvar os dados.<br>O sistema deve validar todos os dados inseridos, impedindo o salvamento de campos essenciais vazios, exibindo mensagem de erro abaixo do campo inválido.<br>O sistema deve respeitar a hierarquia lógica no momento do cadastro: Indicador deve estar vinculado à Linha de Ação → Objetivo → Compromisso.<br>Após salvar, o sistema deve exibir: "Dados atualizados com sucesso!" e atualizar a listagem na tela. |
| HU12 | Obrigatória | O sistema deve exibir um dashboard interativo com gráficos referentes aos indicadores, acessível a todos os usuários.<br>O usuário deve poder selecionar um problema público por vez e visualizar todos os gráficos dos indicadores relacionados.<br>Os dados dos indicadores devem ser desenhados em componentes visuais legíveis e com legendas e distinção visual entre tipos de indicador.<br>Deve haver uma opção de filtro por "Ano"; ao alterar o ano, o gráfico deve atualizar os dados exibidos. |
| HU13 | Desejável   | Apenas usuários com perfil Conselho Estadual / Secretarias podem realizar a importação de dados de indicadores.<br>O sistema deve aceitar upload de arquivos estritamente nos formatos .csv, .pdf ou .xlsx; arquivos de outros formatos devem ser rejeitados.<br>O sistema deve validar os dados importados, impedindo a inserção de dados inválidos, incompletos ou fora do padrão interno do sistema, informando "Dados inválidos" em caso de erro.<br>Ao finalizar com sucesso, deve exibir: "Importação concluída. Novos registros foram adicionados." |
| HU14 | Obrigatória | O sistema deve permitir que qualquer usuário navegue pela estrutura hierárquica completa do Plano Decenal: Problema público → Compromisso → Objetivo → Linhas de Ação → Tarefas.<br>A navegação deve ser intuitiva e permitir acesso a cada nível da hierarquia de forma organizada. |
| HU15 | Importante  | O sistema deve exibir, no detalhamento de cada Linha de Ação, o Responsável Principal e as entidades Colaboradoras.<br>Qualquer usuário deve conseguir visualizar essas informações sem necessidade de login. |
| HU16 | Importante  | A tela de logs de auditoria deve ser acessível estritamente aos perfis Administrador e Gestor SEDEF.<br>Cada log deve conter obrigatoriamente: data/hora do evento, identificação do usuário (id/e-mail), tipo de ação (adição, alteração ou remoção) e identificação do dado alterado (tarefa ou indicador).<br>Os registros de log não podem ser editados ou apagados por nenhum usuário do sistema.<br>O administrador ou gestor SEDEF deve conseguir filtrar os logs por período (data inicial e final), por tipo de ação e por usuário. |
| HU17 | Obrigatória | Após o cadastro de um novo usuário pelo administrador, o sistema deve enviar automaticamente para o e-mail cadastrado as credenciais de acesso (e-mail e senha inicial).<br>O e-mail deve ser enviado sem necessidade de ação adicional por parte do administrador. |
| HU18 | Obrigatória | No primeiro login com as credenciais definidas pelo administrador, o sistema deve obrigar o usuário a definir uma nova senha particular antes de acessar qualquer outra funcionalidade.<br>O sistema deve validar a nova senha conforme os critérios de segurança definidos.<br>Após a troca, o usuário deve conseguir acessar o sistema normalmente com a nova senha. |
| HU19 | Desejável   | Deve existir uma opção visível em qualquer parte do sistema para ativar a tradução em Libras.<br>Ao selecionar com o mouse um elemento de texto da interface, um avatar virtual deve realizar a tradução em tempo real.<br>A funcionalidade deve estar disponível a qualquer usuário, sem necessidade de login. |
| HU20 | Obrigatória | O sistema deve permitir que somente o perfil Administrador ou Gestor SEDEF cadastre novos usuários.<br>O formulário de cadastro deve exigir os campos: E-mail, Nome e Perfil (Gestor SEDEF, Conselho Estadual / Secretaria ou Administrador).<br>Usuários sem permissão não devem acessar essa funcionalidade. |
| HU21 | Importante  | O sistema deve exibir, em cada indicador, a data e a hora da última atualização, o tipo de ação (adição, remoção ou alteração) e o nome do usuário responsável pela atualização.<br>Qualquer usuário, incluindo o público geral, deve conseguir visualizar essas informações sem necessidade de login. |
