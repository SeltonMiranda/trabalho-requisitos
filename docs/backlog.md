| ID   | Prioridade  | Critérios de Aceitação                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| HU01 | Obrigatória | O sistema deverá exibir os 23 problemas públicos cadastrados.<br>Cada problema público deverá estar vinculado ao respectivo Compromisso Decenal da Criança e do Adolescente.<br>O usuário deverá conseguir acessar essas informações sem realizar login.<br>As informações deverão ser apresentadas de forma organizada e compreensível.                                                                                                                                  |
| HU02 | Obrigatória | O sistema deverá solicitar e-mail e senha para autenticação.<br>Usuários sem autenticação não deverão acessar áreas restritas.<br>O sistema deverá identificar o perfil do usuário após o login.<br>O acesso deverá respeitar os perfis definidos no sistema: Administrador, SEDEF, Conselho Estadual/Secretarias e público geral.<br>Caso as credenciais estejam incorretas, o sistema deverá exibir uma mensagem de erro.                                              |
| HU03 | Obrigatória | O sistema deverá permitir filtro por compromisso.<br>O sistema deverá permitir filtro por indicador.<br>O sistema deverá permitir filtro por período de tempo.<br>O sistema deverá permitir filtro por responsável.<br>Os filtros deverão poder ser aplicados de forma cumulativa.<br>Os resultados exibidos deverão ser atualizados de acordo com os filtros selecionados.                                                                                               |
| HU04 | Obrigatória | O administrador deverá conseguir visualizar os usuários cadastrados.<br>O administrador deverá conseguir alterar o nível de permissão dos usuários.<br>O administrador deverá conseguir elevar ou reduzir permissões de acesso.<br>Usuários sem permissão não deverão conseguir acessar o gerenciamento de usuários.<br>As alterações realizadas nas permissões deverão ser registradas pelo sistema.                                                                     |
| HU05 | Obrigatória | O usuário deverá ter feito uma busca pelas opções de filtro.<br>O sistema deverá permitir que o usuário selecione o formato da exportação.<br>O sistema deverá inicializar o download automático no dispositivo do usuário com os dados filtrados no formato selecionado para exportação.                                                                                                                                                                                |
| HU06 | Importante | O sistema deverá verificar se o usuário está autenticado.<br>O sistema deverá respeitar os perfis definidos no sistema: Administrador, SEDEF, Conselho Estadual/Secretarias e público geral.<br>Caso as credenciais estejam incorretas, o sistema deverá retornar uma mensagem de erro.<br>O histórico deverá ser exibido em uma lista ordenando as atualizações das mais recentes para as menos recentes, com informações de data, hora e responsável pelas alterações. |
| HU07 | Obrigatória | O sistema deverá exibir no formato DD/MM/AAAA - HH:MM a data da mais recente atualização nos dados da tarefa do sistema.                                                                                                                                                                                                                                                                                                                                                |
| HU08 | Importante  | O sistema deve apresentar um campo de busca textual para "Palavra-chave" e opções para "Responsável", "Eixo do Plano Decenal" e "Compromisso".<br>O usuário deve conseguir combinar dois ou mais filtros simultaneamente para refinar a busca.<br>Nenhum dos campos de busca deve ser obrigatório; se o usuário clicar em "Buscar" com tudo em branco, o sistema deve listar todos os registros.<br>Caso nenhum registro atenda aos critérios, o sistema deve exibir: "Nenhum resultado encontrado para os filtros aplicados." |
| HU09 | Desejável   | Deve existir uma opção para ativar ou desativar o alto contraste.<br>Ao ativar, o fundo do sistema deve mudar para uma cor escura e os textos para cores de alta visibilidade (branco ou amarelo).<br>Se o usuário navegar entre as páginas com o contraste ativo, o modo deve se manter ativado.                                                                                                                                                                        |
| HU10 | Obrigatória        | Apenas usuários autenticados com o perfil "Conselho Estadual" ou "Secretaria" podem visualizar o formulário e salvar esses dados.<br>O sistema deve respeitar a hierarquia lógica no momento do cadastro: uma Linha de Ação deve estar vinculada a um Objetivo, que por sua vez se vincula a um Compromisso.<br>O sistema não deve permitir o salvamento de campos essenciais vazios; deve exibir uma mensagem de erro abaixo do campo inválido.<br>Após salvar, o sistema deve exibir: "Dados atualizados com sucesso!" e atualizar a listagem na tela. |
| HU11 | Importante  | A tela/relatório de logs de auditoria deve ser estritamente restrita aos perfis "Administrador" e "SEDEF".<br>Cada log gerado deve conter obrigatoriamente: Data/Hora do evento, Identificação do Usuário (id/e-mail), Tipo de Ação (Criação, Adição ou Remoção) e Identificação do dado alterado.<br>Os registros de log de auditoria não podem ser editados ou apagados por nenhum usuário do sistema.<br>O administrador deve conseguir filtrar os logs por período (Data Inicial e Final), por tipo de ação e por usuário. |
| HU12 | Obrigatória        | Os dados dos indicadores devem ser desenhados em componentes visuais legíveis e com legendas.<br>Deve haver uma opção de "Ano" na tela; ao alterar o ano, o gráfico deve atualizar os dados após o clique.                                                                                                                                                                                                                                                              |
| HU13 | Desejável   | O sistema deve aceitar o upload de arquivos estritamente nos formatos .csv, .pdf ou .xlsx; arquivos de outros formatos devem ser rejeitados.<br>O sistema deve validar a planilha inteira; se houver algum erro, deve informar com a mensagem "Dados inválidos".<br>Ao finalizar com sucesso, deve exibir: "Importação concluída. Novos registros foram adicionados."                                                                                                     |
